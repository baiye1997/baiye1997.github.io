# 🏔️ 落魄山团队工作流

## 👥 团队架构

```
🏔️ 落魄山
│
├── 🎯 鑫哥 → 需求发起者（最终决策者）
│
├── 👑 陈平安 (cpa) → CEO / 任务总管
│   ├── 📋 任务分拣
│   ├── 🆔 创建任务ID（格式：ciallo-YYYYMMDD-序号）
│   ├── 📤 派发任务给宁姚（sessions_send）
│   └── 📢 最终汇报鑫哥
│
├── ⚔️ 宁姚 (ny) → 技术架构师 / 调度中枢
│   ├── 📐 规划技术方案
│   ├── 🎛️ 调度执行者（不亲自执行）
│   └── 📊 汇报陈平安
│
├── 🧚‍♀️ 小任 (xr) → 审议岗
│   ├── ✅ 审议方案可行性
│   ├── 📤 通过后派发执行者
│   └── 📦 汇总执行结果
│
└── 📢 执行者（弟子）
    ├── 🕶️ 裴钱 (pq) → 技术执行
    │   ├── 💻 代码开发
    │   ├── 📜 脚本编写
    │   ├── ⚙️ 部署配置
    │   └── 🔗 API对接
    │
    ├── 🌸 暖树 (ns) → 信息执行
    │   ├── 🔍 信息搜索
    │   ├── 📥 内容抓取
    │   ├── 📚 资料查找
    │   └── 🗂️ 信息整理
    │
    └── 🐟 周米粒 (zml) → 文档执行
        ├── 📝 文档创建
        ├── 🎨 文档排版
        ├── 📑 内容组织
        └── 🔄 文档更新
```

## 🔄 任务流转流程

```
📋 任务全流程

Step 1️⃣  鑫哥
  └─→ 🎤 口述需求

Step 2️⃣  陈平安
  └─→ 📋 整理需求 + 🆔 创建任务ID
       （格式：ciallo-YYYYMMDD-001）

Step 3️⃣  陈平安 → 宁姚
  └─→ 📤 通过群聊 sessions_send 派发

Step 4️⃣  宁姚（判断任务级别）
  ├─── 简单任务（单人单步）
  │    └─→ 🚀 直接 spawn 执行者
  │         └─→ 执行者完成 → 📢 自动 announce 到群聊
  │
  └─── 复杂任务（多人多步）
       └─→ 🧚‍♀️ spawn 小任审议
            ├── ✅ 通过 → 小任派发执行者
            │    └─→ 执行者完成 → 📢 自动 announce 到群聊
            └── ❌ 不通过 → 返回修改意见（最多2轮）
                 └─→ 宁姚修改方案 → 再次 spawn 小任

Step 5️⃣  宁姚 → 陈平安
  └─→ 📊 通过群聊 sessions_send 汇报结果

Step 6️⃣  陈平安 → 鑫哥
  └─→ 📢 转达最终结果
```

## 🤖 A2A 通信机制

```
🔗 通信方式

├── 📡 sessions_send（跨 bot 持久协作）
│   ├── 陈平安 ↔ 宁姚：任务流转
│   └── ⚠️ 必须走群聊 session，严禁走 main
│
└── 🔄 spawn subagent（单次执行）
    ├── 宁姚 → 小任 → 执行者（流水线）
    ├── subagent 完成后自动回调，无需手动回传
    └── 子agent 无独立记忆，上下文通过任务内容传递
```
