# 落魄山 Obsidian 归档规范

## 概述

本文档定义了落魄山 Obsidian 知识库中所有文档的归档标准，确保格式一致、可检索、可关联。

---

## 一、YAML Front Matter 规范

所有文档必须在文件开头包含 YAML front matter（由 `---` 包裹）。

### 必填字段

| 字段 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| `date` | 日期 | 文档日期（YYYY-MM-DD） | `2026-04-23` |
| `tags` | 数组 | 标签列表 | `[山水邸报]` |
| `type` | 字符串 | 文档类型 | `早报` / `调研报告` / `投资分析` |

### 可选字段

| 字段 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| `weather` | 字符串 | 天气概要（早晚报专用） | `多云 16°C-26°C` |
| `source` | 字符串 | 数据来源 | `花花日记` |
| `aliases` | 数组 | 文档别名 | `["A股行情分析"]` |

### 各空间 Front Matter 示例

**山水邸报（早报/晚报）：**
```yaml
---
date: 2026-04-23 星期四
type: 早报
weather: 多云 16°C-26°C
tags: [山水邸报]
---
```

**调研报告/投资分析：**
```yaml
---
date: 2026-04-02
type: 投资分析
tags: [投资分析, A股, 板块分析]
source: 东方财富网
---
```

## 二、标签规范

### 基础标签

- 年月标签：`2026年4月`（对应文档数据月份）
- 领域标签：根据内容选择，如 `A股`、`基金`、`债券`

### 文档类型专用标签

| 文档类型 | 标签 |
| :--- | :--- |
| 每日早晚报 | `山水邸报` |
| 投资分析类 | `投资分析` |
| 调研报告类 | `调研报告` |
| 周报 | `周报` |

### 其他常用标签

- 按主题：`板块分析`、`资金流向`、`市场走势`、`宏观经济`
- 按市场：`A股`、`港股`、`美股`、`黄金`、`债券`
- 按状态：`盘中`、`收盘`、`周报`、`月报`

## 三、文件名规范

### 格式

```
YYYY-MM-DD-标题.md
```

### 示例

- `2026-04-23-早报-新闻速递.md`
- `2026-04-02-A股行情与持仓分析.md`
- `2026-04-07-陈平安每周清理脚本完善.md`

### 注意事项

- 日期为文档数据日期，非归档日期
- 标题使用中文，简洁明了
- 文件名中不要包含特殊字符（引号、斜杠等）

## 四、文档结构规范

### 标题层级

| 层级 | 用途 | 示例 |
| :--- | :--- | :--- |
| H1（`#`） | 文档标题（唯一） | `# 🏔️ 山水邸报 · 早报` |
| H2（`##`） | 一级段落/章节 | `## 市场概况`、`## 今日新闻` |
| H3（`###`） | 子章节 | `### AI · 大模型` |

### 标准结构模板

```markdown
# [文档标题]

> **YYYY年M月D日 · 星期X**　　*知行合一，格物致知*

---

## 章节一

（内容）

## 章节二

（内容）

---

*由 Hermes 整理生成*
```

## 六、关联规范

### Wiki-Link 格式

- 同文件夹内：`[[文件名]]`
- 跨文件夹：`[[文件夹/文件名|显示文字]]`
- 表格内：`[[文件夹/文件名]]`（避免 `|` 被解析为列分隔符）

### README 索引

每个文件夹的 README.md 负责：
- 列出该文件夹下所有笔记（降序排列）
- 使用 wiki-link 关联每篇笔记
- 提供返回首页和关系图谱的链接

### 关系图谱

`graph.md` 展示整个知识库的目录结构和笔记关联，每次新增笔记后需同步更新。

## 七、检查清单

创建/归档文档时，对照以下清单：

- [ ] 文件开头有 `---` 包裹的 YAML front matter
- [ ] 包含必填字段：`date`、`tags`、`type`
- [ ] 文件名格式：`YYYY-MM-DD-标题.md`
- [ ] H1 标题唯一且与文档内容对应
- [ ] 正文从 H2 开始分段
- [ ] 如有需要，添加 wiki-link 关联相关笔记
- [ ] 更新对应文件夹的 README.md
- [ ] 执行 `ob sync` 同步到 Obsidian 云端
